How this was built — end-to-end process

1

Data mining & cleaning

Pulled data from multiple sources (tool logs, maintenance records, MES). Cleaned and standardized formats, handled missing values, and aligned tool IDs and timestamps so everything could be joined and filtered consistently.

2

Data analysis

Explored the data to define the right KPIs: what to track for maintenance compliance, cost drivers, and EPA reporting. Identified gaps (e.g. unmaintained tools) and validated definitions with operations before building reports.

3

SQL pipelines & automated dashboards

Built SQL pipelines to aggregate and transform the cleaned data on a schedule. Connected Power BI to those pipelines so dashboards refresh automatically—single source of truth for tool status, maintenance KPIs, and cost trends.
